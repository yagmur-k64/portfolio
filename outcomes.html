Leeruitkomsten

Ik heb deze website gemaak voor mijn studie. De opdracht was om een digitale portfolio te maken. In deze portfolio moet ik een aantal leeruitkomsten kunnen tonen. Omdat ik Associate Degree doe heb ik 4 leerdoelen, namelijk interactive media, development, iterative design en professional standard. Hier onder zijn beschrijvingen van elk leerdoel te vinden samen met bewijsmateriaal en een reflectie.

Reflectie

Ik vind onderzoek doen het saaiste aan ontwerpen. Maar toch doe ik wel altijd diep onderzoek naar het onderwerp waar ik aan werk. Dit heb ik vooral bij projectX goed aan kunnen tonen. Ik heb concurrentie onderzoek gedaan en ik heb iemand uit de doelgroep geïnterviewd. We hebben bij de Fix that UI project user tests gedaan bij mensen van verschillende leefitjds categorieën. Verder heb ik zelf geen officieel test gedaan. Ik heb mijn websites vaak aan anderen laten zien. Hierbij heb ik mentale notities gemaakt van de dingen waar ze tegenaan liepen. Deze problemen heb ik daarna meteen opgelost, zoals de footer in mijn projectX.
Op het punt van trends en ontwikkelingen denk ik dat ik heel goed heb gedaan. Omdat ik altijd eerst doelgroep onderzoek doe kan ik me daar op aanpassen. Bij projectX ging het heel makkelijk voor mij omdat ik zelf 7 jaar lang bij dat bedrijf heb gewerkt. Hierdoor wist ik precies wat er nodig was en hoe de doelgroep zou zijn. Voor deze portfolio website is de doelgroep mijn docenten. Ik heb een aantal keer feedback gevraagd over de opmaak. Mijn coach heeft mij uitgelegd hoe hij alles wilt zien. Ik heb het dus niet ingewikkelder gemaakt dan dat het nodig is.
Ik vind dus dat ik deze leerdoel goed heb kunnen aantonen. Zoals hierboven bij de bewijsmateriaal ook te zien is, heb ik deze doel meerdere keren kunnen aantonen.

Reflectie

Ik was aan het begin van dit semester heel erg onwetend over development, maar nu kan ik met trots zeggen dat daar een verandering in is gekomen. Ik heb vorige semester een website gemaakt waarin ik heel veel AI had gebruikt. Ik heb vaak terug gekeken naar die code om het toe te kunne passen. Dat heeft mij heel erg geholpen samen met de inspecteren functie. Sinds de development project voor HikeOne vind ik dat ik veel beter hier in ben geworden. Ik heb veel trucjes geleerd van zowel mijn collega's als het internet. Ik heb op het begin van deze portfolio website veel hulp gekregen, maar richting het einde heb ik eigenlijk alles zelf gedaan. Ook Javascript snap ik steeds meer en meer naarmate ik het vaker gebruik.
Ik ben super trots op mijzelf voor wat ik heb geleverd voor projectX. Ik moet alleen nog oefenen met het responsief maken. Ik heb tijdens projectX GitHub ook heel goed gebruikt. Daar zijn alle stappen terug te vinden. Ik vind dus dat ik deze leerdoel echt wel heb behaald. Ik ben zeker vooruit gegaan in mijn codeel skills.

Reflectie

Ik heb in alle projecten veel aandacht besteed aan itereren. Dit heb ik geleerd bij de Product Ontwerpen opleiding die ik heb gedaan. Daar moesten we iets heel vaak itereren en testen voordat we verder mochten. Alles documenteren gaat niet altijd zo goed bij mij. Omdat ik zo erg in de flow zit, vergeet ik heel vaak om tussendoor screenshots te maken. Ik vind dat ik dat wel goed heb gedaan deze semester.
Ik heb al veel ervaring met meerdere ontwerp tools. Heel de brandguide van Blik heb ik in Indesign gemaakt. De logo's heb ik met Illustrator gemaakt. Ik heb in de eerste project veel mijn Adobe skills goed kunnen tonen en verbeteren. In de andere drie projecten heb ik vooral met Figma gewerkt. Mijn Figma vaardigheden zijn hierdoor ook flink vooruit gegaan. In de UI project heb ik veel geoefend met components. Ik heb het nu helemaal onder de knie.
Aan het einde van dit semester denk ik dat ik mijn leerdoel voor Iterative Design heb behaald. Ik heb meerdere keren laten zien dat ik dit kan. Ik heb altijd complimenten gekregen van mijn medestudenten en docenten over mijn design skills.

Reflectie

Ik dacht vroegel altijd dat mijn professional skills slecht waren. Maar ik ben er in deze periode achter gekomen dat het helemaal niet zo slecht is als dat ik dacht. Ik heb heel vaak de leiding genomen in de groepsopdrachten. Dit kwam meerendeels omdat iedereen in mijn groep heel lui en niet serieus was. Hierdoor moest ik naar voren stappen en de leiding nemen. Ik was meestal degene die een planning ging maken met de groep. In de tweede project hadden we twee project leiders gekozen, maar na een weekje zag ik al dat daar niks van zou gebeuren dus heb ik het op me genomen om iedereen aan de lijn te houden.
Ik heb tijdens de portfolio review de opmerking gekregen dat ik vaker feedback moet vragen en deze moet documenteren in FeedPulse. Dit is nogsteeds een punt waar ik aan moet werken. Ookal vraag af en toe om korte feedback, vergeet ik soms om een FeedPulse te vragen. Ik vraag wel heel vaak peer feedback. Ik laat mijn werk zien en controlleren aan andere studenten en vrienden.
In de projecten van Blik en HikeOne hebben we niet vaak contact opgenomen met de klanten. In mijn individuele project heb ik dit wel gedaan. Ik heb Op het begin een gesprek gehad waarin mijn klant mij heeft verteld wat ze willen hebben. Toen ik eenmaal aan het werk was heb ik regelmatig updates gestuurd om feedback van ze te krijgen. Ik denk dat dit ook fijn is voot hun omdat ze dan weten dat er ook daadwerkelijk tijd en moeite in wordt gestoken.

Ik vind dat ik deze leerdoel goed heb kunnen tonen aan de hand van de bovenstaande screenshots en PDF bestanden. Mijn groep is zeer tevreden met mijn werk, dit hebben ze vaak gezegd. Ik heb altijd positief feedback gekregen van ze. Ik heb uiteindelijk geleerd dat het heel belangrijk is om in contact te blijven met de belanghebbenden.
